# Strategy Pattern — Real/Mock Client Switching

This reference explains the Strategy pattern implementation for switching
between real and WireMock-backed clients. Actual code is generated by
the language track skills in their Layer 2 references.

---

## Concept

```
«interface»
ApiClientStrategy
    + get(endpoint) → Response
    + post(endpoint, body) → Response
    + put(endpoint, body) → Response
    + delete(endpoint) → Response
           ▲
           │ implements
    ┌──────┴──────┐
    │             │
RealApiClient  MockApiClient
(hits live API) (hits WireMock at localhost:8080)
           ▲
           │ creates
     ClientFactory.build(TEST_MODE)
```

---

## Switching Between Real and Mock

The strategy is controlled by a single environment variable:

```bash
TEST_MODE=mock   # tests run against WireMock stubs (fast, isolated)
TEST_MODE=real   # tests run against real API (integration confidence)
```

To run smoke tests against real API and regression against mock:

```bash
# CI — fast regression suite:
TEST_MODE=mock pytest layer_4_tests/ -m "regression" -n auto

# CI — integration validation:
TEST_MODE=real pytest layer_4_tests/ -m "smoke"
```

Zero test code changes. The factory handles the switch.

---

## For UI Tests: Strategy Applied at the Application Level

UI tests don't switch client strategies — the browser uses a real HTTP
connection to the application. The "mock strategy" for UI tests is
controlling which backend URL the application uses:

```bash
# Real backend:
AUT_BASE_URL=http://localhost:3000
API_BASE_URL=https://real-api.staging.com    # app config
TEST_MODE=real

# Mock backend (WireMock):
AUT_BASE_URL=http://localhost:3000
API_BASE_URL=http://localhost:8080           # app points at WireMock
TEST_MODE=mock
```

The application must support a configurable backend URL at startup.
This is standard for any well-architected frontend application.

## Extending: Adding a Prism Client

When adding Prism as an alternative mock, the Strategy pattern makes it
trivial — just add a new implementation:

```python
class PrismApiClient(BaseApiClient):
    """Points at Prism mock server (OpenAPI-driven)."""
    def __init__(self, token=None):
        super().__init__(token=token)
        self.base_url = "http://localhost:4010"   # Prism default port
```

```python
# Update ClientFactory:
if mode == "prism":
    return PrismApiClient(token=token)
```

Adding a new mock tool requires:
1. New client class (5 lines)
2. One new branch in ClientFactory (2 lines)
3. Zero changes to any test, service, or model file
